# Day 9 – AI-Powered Application Enhancement (NutriScope)

## Objective

Learn how iterative prompting can improve an AI-generated application by building an MVP first and then enhancing it with additional features.

---

# Task Performed

Today I used Claude to build and enhance **NutriScope**, an AI-generated nutrition tracking application.

The workflow included:

1. Generating the MVP version using Prompt 1.
2. Downloading the generated HTML application.
3. Testing the application's core functionality.
4. Using Prompt 2 to enhance the existing application.
5. Generating an improved version with advanced features.
6. Comparing the MVP and enhanced versions.
7. Documenting observations and key learnings.

---

# MVP Version Features

The initial version of NutriScope included:

- User profile inputs (Age, Gender, Height, Weight, Activity Level, Dietary Preference)
- Food logging system
- Food database with 20 common foods
- Calorie and macronutrient tracking
- Micronutrient tracking
- Energy and nutrition dashboard
- Nutritional recommendations
- Responsive dark-themed UI
- Single HTML application

---

# Enhanced Version Features

The enhanced version introduced several improvements:

- CSV food data upload
- Expanded food database (+40 foods)
- Additional micronutrient tracking
- Two-day meal planner
- Nutrition risk analysis
- Educational disclaimer
- Nutrition information sources
- Improved charts and visualizations
- Advanced personalized recommendations

---

---

# Testing

I tested both versions by:

- Logging food entries.
- Verifying nutrition calculations.
- Exploring dashboard charts.
- Comparing recommendations.
- Checking responsiveness across different screen sizes.

---

# Key Learnings

- AI can generate functional web applications from detailed prompts.
- Iterative prompting significantly improves application quality.
- Building an MVP first and refining it is an effective development approach.
- Well-structured prompts produce more complete and maintainable applications.

---

# My Reflection

Today's exercise demonstrated how AI can support rapid application development through iterative improvements. Instead of attempting to build the final product in one step, generating an MVP first and enhancing it with additional prompts resulted in a more complete, feature-rich, and user-friendly application.
