# 30-Day Job-Readiness Roadmap
**For:** 2nd-year CS student | DSA + basic dev skills | Goal: internship/job in 6 months | 5-6 hrs/day

---

## The strategy

You already have DSA fundamentals and basic dev knowledge — so this month isn't "learn to code from zero," it's **turn what you know into something a recruiter can see and test**. Two tracks run in parallel every day:

- **Track A (2 hrs/day): DSA maintenance** — you don't stop solving problems, you sharpen aim toward interview patterns.
- **Track B (3-4 hrs/day): Ship one real, deployed full-stack project** — because "I know React" doesn't get callbacks, a live link does.

By day 30 you'll have a **deployed full-stack app + polished GitHub + resume-ready bullet points + 90+ problems solved with pattern recognition**, not just "watched."

---

- **Task manager / Kanban board** (like a mini Trello) — great because it touches CRUD, auth, and state management, and is easy to explain in interviews
- **Or:** a **notes/collab app** — since you've already built CollabAI, a scoped-down version (single workspace, real-time optional) shows depth without redoing everything from scratch

## Final Outcome (Day 30)

By the end of this roadmap you will have:
1. **One fully deployed, full-stack project** with auth, database, and a real feature — live link + clean GitHub repo + demo video
2. **~90-100 DSA problems solved** across arrays, strings, linked lists, trees, graphs, and intro DP — with pattern recognition, not just memorized solutions
3. **An updated resume and LinkedIn** with a concrete project to talk about
4. **10-25 applications sent** to internship/entry-level roles matching frontend, backend, or UI/UX
5. **Interview-ready self-introduction and project pitch**, rehearsed at least twice

This is Month 1 of 6. Months 2-3: repeat this cycle with a second, more ambitious project (or contribute to open source) + push DSA into medium/hard territory. Months 4-6: heavy interview prep, mock interviews, and high-volume applications.

# 30-Day DSA & Competitive Programming Roadmap

**Goal:** Build a strong, pattern-based foundation in Data Structures & Algorithms and get comfortable with competitive programming contests — in 30 days, ~2-3 hours/day.

**Assumption:** You already know basic programming (loops, functions, arrays) in at least one language (C++/Java/Python). This roadmap is beginner-friendly *for DSA*, not for programming itself.

---

## How to Use This Roadmap

- Each week has a **milestone** (the skill you should walk away with).
- Each day has a **task** — usually a topic + 3-5 problems.
- Solve problems on **paper/whiteboard first**, then code. Don't jump to solutions before 25-30 min of genuine effort.
- Keep a **mistakes log**: every problem you get wrong, note *why* (wrong approach, edge case, silly bug).

---

## Final Outcome (After 30 Days)

By Day 30, you should be able to:
- Explain and apply time/space complexity to any solution you write
- Implement arrays, linked lists, stacks, queues, trees, and graphs from scratch
- Recognize and apply two pointers, sliding window, BFS/DFS, and basic DP patterns
- Comfortably solve LeetCode Easy problems and most Medium problems within 20-30 minutes
- Have completed at least one live competitive programming contest
- Have 4 small projects for your portfolio: word frequency analyzer, browser history simulator, maze solver, and a personal DSA tracker

**Next steps after Day 30:** Move into advanced DP, advanced graph algorithms (Dijkstra, Union-Find, MST), and start targeting Codeforces rating 1200+ / LeetCode Medium-Hard consistently.
